# VibeOs — План реализации

Поэтапный план. Каждый этап заканчивается работающим, демонстрируемым результатом
(walking skeleton → наращивание). Чекбоксы для трекинга.

Принцип: **сначала "оконность" с фейковым контентом, потом подключаем агента.**
Это позволяет отлаживать window manager без затрат на токены.

---

## M0 — Скелет проекта  ✅ ГОТОВО
Цель: репозиторий, сборка, пустой рабочий стол открывается в браузере.
- [ ] `git init`, базовый `package.json`, workspaces (`shell`, `server`).
- [ ] Vite-проект `shell/` → отдаёт пустую страницу с wallpaper + menu bar + dock.
- [ ] Node-проект `server/` → поднимается, отвечает на `GET /health`.
- [ ] README с командами запуска (`npm run dev` поднимает shell+server).
- **DoD:** открыл localhost — видно рабочий стол macOS-стиля (статичный), без окон.

## M1 — Window Manager (без AI)  ✅ ГОТОВО
Цель: полноценная "оконность" на захардкоженном HTML.
- [ ] Модель `WindowState` + стор окон.
- [ ] Создание окна с рамкой, titlebar, traffic lights, телом-iframe (srcdoc =
      статичная заглушка).
- [ ] **Drag** за titlebar; **resize** за края/углы (8 хэндлов).
- [ ] **Focus / z-index**: клик поднимает окно, подсветка активного.
- [ ] **Close / minimize(в dock) / maximize(restore)**.
- [ ] Клик по иконке в dock открывает окно с заглушкой.
- **DoD:** можно открыть несколько окон, таскать/ресайзить/закрывать, фокус работает.

## M2 — Design System  ✅ ГОТОВО
Цель: библиотека UI, в которой будут собираться приложения.
- [ ] `design-system.css`: переменные (цвета/тени/радиусы/SF Pro) + компоненты
      (`.vibe-btn`, toolbar, list, input, card, sidebar, table, tabs, segmented…).
- [ ] Набор иконок/ассетов (dock-иконки, glyph-набор, обои).
- [ ] **Storybook-страница** (статичная демо `design-system/preview.html`) со всеми
      компонентами — глазами проверить вид.
- [ ] `ds-prompt.md` — компактное текстовое описание DS (классы + правила + 2-3
      few-shot примера разметки) для системного промпта.
- **DoD:** заглушки окон из M1 выглядят как настоящие macOS-приложения.

## M3 — Agent Bridge + Claude Code  ✅ ГОТОВО
Цель: backend генерирует HTML через Claude Code (пока без интерактива внутри окон).
- [ ] WebSocket-канал shell ↔ server (протокол из architecture.md §4.2).
- [ ] `claude-runner.ts`: интеграция через **Claude Agent SDK** (фолбэк CLI
      headless), стриминг, инструменты отключены.
- [ ] `prompt-builder.ts`: системный промпт (`prompts/system.md` + `ds-prompt.md`) +
      бриф приложения (`prompts/apps/<app>.md`).
- [ ] `sanitizer.ts`: вырезать markdown-заборы, allowlist тегов/классов, DOMPurify.
- [ ] Поток `launch`: dock → server → стрим HTML → рендер в iframe (+ skeleton).
- [ ] Инъекция iframe-обёртки (design-system.css) в `srcdoc`.
- **DoD:** клик по "Notepad" в dock → агент генерит реальный UI заметочника в окне.

## M4 — Интерактив: события из iframe  ✅ ЯДРО ГОТОВО (проверить в браузере)
Цель: нажатия внутри окон управляют приложением через агента.
- [ ] `bridge.js` (iframe-shim): перехват `data-action`-кликов/форм, сбор стейта
      формы, `postMessage` в shell.
- [ ] Shell принимает `vibe-event` → шлёт `event` в server с `sessionId` окна.
- [ ] `event`-поток: resume сессии → промпт с событием+стейтом → новый HTML →
      ре-рендер iframe.
- [ ] Индикация "thinking" на окне/кнопке во время генерации.
- **DoD:** в Notepad набрал текст, нажал Save → агент обновил UI (напр. показал
      "сохранено" + список заметок). Калькулятор считает по кликам.

## M5 — Сессии и состояние приложений  ⏱ ~1 день
Цель: приложения помнят свой стейт, layout восстанавливается.
- [ ] `session-store.ts`: 1 окно = 1 Claude-сессия; маппинг windowId↔sessionId.
- [ ] Закрытие окна → завершение сессии; (опц.) персист на диск.
- [ ] (Опц.) Восстановление открытых окон и их геометрии после перезагрузки.
- **DoD:** приложение сохраняет контекст между действиями; повторное действие
      учитывает прошлые.

## M6 — Производительность и кэш  ⏱ ~1 день
Цель: быстрее и дешевле.
- [ ] `cache.ts`: кэш стартовых экранов и повторяющихся (appId+event-хэш) рендеров.
- [ ] Прогрессивный стрим в iframe (рендер по чанкам, а не по `done`).
- [ ] (Опц.) Targeted DOM-patch вместо full re-render (агент возвращает изменения),
      чтобы сохранять фокус/скролл.
- **DoD:** типовые экраны открываются мгновенно из кэша; ввод не теряет фокус.

## M7 — Набор приложений + полировка  🔄 В ПРОЦЕССЕ
Цель: "ОС ощущается настоящей".
- [x] Анимации окон: open (scale+fade), close, minimize-в-dock, restore.

- [x] Menu bar: имя активного приложения (обновляется по фокусу/meta/закрытию).
- [x] Обработка ошибок агента: оверлей с кнопкой Retry (регенерация).
- [ ] (опц.) Рабочие выпадающие меню (File/Edit/Window) активного приложения.
- [ ] (опц.) Звук, контекстные меню, genie-анимация по координатам дока.
- [ ] (опц.) Курируемые брифы dock-приложений для более богатых дефолтов.
- [ ] (опц.) Демо-сценарий + запись.
- **DoD:** можно "пользоваться" несколькими приложениями, всё консистентно и живо.

## M8 — Spotlight + Launchpad + App Cache  ✅ ГОТОВО
Цель: запуск любого приложения по имени, переиспользование сгенерированных.
- [x] **App Cache** на backend: хранение `{key, brief, name, glyph, category,
      html, ts, opens}` с persist (JSON `server/.cache/apps.json`); `/apps` endpoint.
- [x] Агент при первой генерации возвращает метаданные приложения (имя, glyph,
      категория) → запись в App Cache. Кэш-хит сервится мгновенно; первое
      взаимодействие с кэш-окном поднимает свежую сессию из брифа.
- [x] **Динамический App Registry**: dock + кэшированные приложения из одного
      источника (`apps-api.getAllApps`).
- [x] **Spotlight (⌘K + иконка-поиск)**: оверлей-поиск; ввод любого имени → генерация.
- [x] **Автоподстановка**: фильтр App Cache + встроенных (точное → префикс → fuzzy),
      кэшированные выше; пункт «Generate '<запрос>'» если нет совпадений.
- [x] Повторный запуск кэшированного приложения — **мгновенно из кэша**.
- [x] **Launchpad**: полноэкранная сетка всех приложений (встроенные + кэш), поиск,
      запуск кликом; иконка 🚀 в доке.
- [x] **⌘J** — сброс кэша активного приложения и регенерация (force).
- **DoD:** ⌘K → ввёл "Pomodoro timer" → сгенерилось и попало в кэш; повторный
      ввод "pomo…" мгновенно подставляет его; оно же видно в Launchpad. ✅

## Порядок и зависимости
```
M0 → M1 → M2 → M3 → M4 → M5 → M6 → M7 → M8
              (M2 нужен для M3; M3 — для M4; M8 опирается на M3+M6)
```
M1+M2 можно вести параллельно (разные люди/сессии): WM и DS независимы.

## Зафиксированные решения (2026-06-06)
1. **Shell-фреймворк:** ✅ Vanilla TS + Vite.
2. **Интеграция агента:** ✅ Claude Agent SDK (`@anthropic-ai/claude-agent-sdk`),
   фолбэк CLI headless.
3. **Язык UI приложений:** ✅ English (macOS-style).

## Метрики успеха MVP
- Запуск приложения из dock → видимый UI ≤ нескольких секунд (со стримом — мгновенно
  "появляется").
- Нажатия внутри окна корректно меняют приложение через агента.
- Минимум 3 разных приложения убедительно работают.
- Окна: drag/resize/focus/close/min/max без багов.
